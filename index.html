Basics

Here are some basics you need to understand before using our builder:

  • EasyAim uses seconds and milliseconds for timings and other measurements.
  • Scenario maximum duration is 2 minutes (120 seconds).
  • When turning Automatic Assault Rifle on, your fire rate will be 1000ms, meaning you will register 1000 hits every second.
  • The maximum room size is 500 units on all axes.
  • EasyAim uses XYZ axes for directions.
  • X-Axis = Horizontal | Y-Axis = Vertical | Z-Axis = Forward/Backward (negative values mean left/down/backward and positive values mean right/up/forward).
  • Gravity: Negative values = Push the target down | Positive values = Push the target up

Targets

For Targets, we have 4 different options: Cube, Sphere, Capsule (Pill) & Body (Pill + Head).

In Target configuration, you can set: Size, Shape, Health lost on hits, scoring per hit/elimination, maximum health, and the behavior that target will use.

Behaviors

Behaviors are the actions that the bot will execute. These can include spawns, movement, teleports, dashes, or any action you want to create.

In the behavior configuration, you have Behaviors, Frames, and Actions. What does each one do?

Behaviors and Frames are like folders. Imagine you have a game installed on your PC and you want to find its configuration file to change your settings. Most games have folders inside other folders, and the config file will be inside those folders. It's the same with Behaviors, Frames, and Actions.

Behavior is the main folder (parent folder), Frames are the folders inside the behavior, and Actions are the files.

How to Make Static Clicking Scenarios

Creating static scenarios is straightforward. First, create a scenario in the builder and set it to the Clicking category. Then create a target—the best kind of target for this category is Sphere. After creating the target, go to Behaviors, create a behavior → frame → new action, and select "Set Waypoint". Set Waypoint is where you'll configure the bot spawn.

Settings:

  • Teleport = On
  • Prevent Targets Collision = On, set to 2 or 3 for good spacing between targets

After that, you'll need to choose a type of waypoint. For static scenarios, it doesn't matter which one you pick, so choose whatever you prefer. After choosing, you'll set the values for the bot spawn:

  • X = Horizontal spawn values
  • Y = Vertical spawn values
  • Z = Forward/Backward spawn values (if you choose Orbit, Distance is the same as the Z-Axis)

How to Make Dynamic Clicking Scenarios

For dynamic scenarios, the idea is similar, but here we need to use Orbit to create better movement for our targets.

You need to set the spawn to Orbit and also add some new actions to the spawn behavior.

Actions you'll need:

  • Max Speed (controls the target's speed)
  • Rubber Stiffness (controls an invisible line that pulls the target in a direction on XYZ axes)
  • Orbit Restriction (controls the limits for the target to move around)

After setting up all of this, you'll need to create a new behavior for the target's movement. To create movement behaviors, you'll use Set Waypoint, but for movement, you disable the teleport option.

You'll set XYZ axes just like the spawn but for movement, so experiment with values and limits. Also, the higher the rubber stiffness, the more reactive the target will be.

Create 2 frames for movement: Left and Right. On the X-Axis, you'll set negative numbers for left and positive numbers for right.

The distance on movement needs to match the Orbit restriction. If you set them differently, the target won't work correctly. For the Y-Axis, you'll set a range between a negative number and a positive number, for example:

Y-Axis = -15 / 15. This means the bot can move in a range of -15 (down) and +15 (up).

After setting these parameters, all you need to do is test and refine your values and behaviors.

How to Make Ground Tracking Scenarios

For ground tracking, you'll keep the same attributes as dynamic clicking, but now the target model will be a Pill, and you'll set its Y spawn axis to the ground.

After setting it to the ground and changing the model to Pill, we need to ensure the bot won't fly around the map. Go to Max Speed and set Y to 0.

Now all you need to do is adjust values to fit what you want to create—making it fast for reactive tracking, slow and smooth for categories like stability, etc.

How to Make Air Tracking Scenarios

For air scenarios, all you need to do is take the dynamic logic and apply it to a Sphere target with all axis speeds set.

You can make it smooth, reactive, impossible to hit—whatever you want. Just experiment with values to achieve your desired result.
