Static Scenarios

To create static scenarios is pretty simple, all you need to do is create a scenario in the builder then set it to Clicking category, after this you need to create the target the best kind of target for this category is Sphere. After creating the target go to behavior, create a behavior → frame → new action and select "Set Waypoint". Set Waypoint is where you will set up the bot spawn.

Teleport = On |

Prevent Targets Collision On + set it to 2 or 3 for a good distance from each other |

After that you will need to choose a type of waypoint, for static scenarios doesnt matter wich one you will pick so choose whatever you want. After choosing it you will set the values for the bot spawn.

X = Horizontal Spawn values

Y = Vertical Spawn values

Z = Forward / Backward Spawn Values ( if you choose Orbit, Distance is the same as Z Axis).

Dynamic Clicking Scenarios

For Dynamic the idea is the same but here we need to use Orbit to set better movement for our Targets.

Here you need to set the spawn to Orbit but also add some new Actions to the spawn behavior

Actions you will need:

Max Speed (Controls the target's speed)

Rubber Stiffness (Controls an invisible line that pulls the target to the direction)(XYZ)

Orbit Restriction (Controls the limits for the target move around)

After setting up all this you will need to create a new behavior for the targets movement. To Create movement behaviors you will need Set Waypoint but for movement you disable the teleport option.

You will set XYZ Axis just like the spawn but for movement so just try values and limits, also, the higher the rubber stiffness more reactive the target will be.

Make 2 Frames for movement, Left and Right. On X Axis you will set negative number for left and positive number for right.

The distance on movement needs to be the same as the Orbit restriction, if you set it different the target won't work correctly. The Y Axis you will set a range of a negative number and a positive number, for example:

Y Axis = -15 / 15. This means that the bot can move in a range of -15 (down) and +15 (up).

After setting this things all you will need to do is test and improve your values and behaviors.

Ground Tracking Scenarios

For ground tracking you will keep the same attributes of dynamic clicking but now the target model will be a pill and you will set his Y Spawn Axis to the ground.

After setting it to the ground and changing the model to pill we need to make sure that the bot won't fly around the map, so, go on Max Speed and set Y to 0

Now all you need to do is change values to fit what you want to create, making it fast if its reactive,slow and smoother for categories like stability, etc...

Dash Skill

To add a skill like Dashes to a target it’s necessary to understand how EasyAim timings work.

First you need to set a target time like, by default 0 (The target will start the timer at 0 and go up). After setting up the target time you can make your default movement configuration but you need to add one new action called “On Target Time Reached” on your Target’s movement behavior.

After that you will need to create a new behavior for your Dash Skill (or whatever you want to call it). Also create a “reset speed” frame on your movement behavior. How it works:

The bot starts his behavior on time 0, he moves normally like designed but when it hits the time that you set up on “On Target Time Reached” it applies the Dash Skill Behavior. The Dash Behavior needs to be a very rough and fast movement so you’ll set it up with very high speed and rubber stiffness, but after making the dash happen you need the bot to go back to normal, that’s where the “Reset Speed” joins the party.

Reset speed will set the Target’s speed to the default one that you set up on his Spawn Behavior. ( Make sure to set it to the same as the Spawn one ) So after resetting the speed it starts the movement loop again.

Gauntlet Scenarios

Gauntlets are a type of scenario where there’s 2 or more target profiles that run at different times.

First of all we need to make sure that we have a behavior that takes the bot out of the playable area and teleports it outside of it.

Make a behavior called "Spawn Out of Map" and set the spawn to outside of the playable area.

Second, you will need to make different behaviors for each target, as the example above where i called them Bot 1, 2 & 3.

All these behaviors have two “Actions” called “Set Target Time” and “On Target Time Reached” which means that the target has a time where you set it to 0 and it starts counting the Target time from it.

On Target Time Reached means that after the target gets out of 0 to the time you set up on this action it will make another action and here is where you set it to Spawn out of map.

After you complete these steps you will need to just set up the target's initial behavior delay to match what timings you want in your scenario.
